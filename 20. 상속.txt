* 상속 *

뜻
:
쉽게 말해 부모 클래스(상위 클래스)와 자식 클래스(하위 클래스)가 있으며, 
자식 클래스는 부모 클래스를 선택해서, 그 부모의 멤버를 상속받아 그대로 쓸 수 있게 한다.
----------------
이유
:
이미 마련되어 있던 클래스를 재사용해서 만들 수 있기 때문에 효율적이고, 개발 시간을 줄여주게 된다.
-------------------
조건
:
` 부모 클래스의 private 접근 제한을 갖는 필드 및 메소드는 자식이 물려받을 수 없다.
` 부모와 자식 클래스가 서로 다른 패키지에 있다면, 부모의 default 접근 제한을 갖는 필드 및 메소드도 자식이 물려받을 수 없다.
  (default 접근 제한은 ‘같은 패키지에 있는 클래스’만 접근이 가능하게끔 하는 접근 제한자이기 때문)
` 그 이외의 경우는 상속 가능
-------------------
클래스 상속 방법
:
상속받고자 하는 자식 클래스명 옆에 extends 키워드를 붙이고, 상속할 부모 클래스명을 적는다.
자바는 다중 상속을 허용하지 않으므로, extends 뒤에는 하나의 부모 클래스만 와야 한다.
예)
public class parent{....}; // 부모클래스
public class extends parent { .... }; // 자식 클래스

상속
  1. 정의 : super class의 자원(멤버변수, 메서드)를
             sub class가 상속받아 sub class를 정의한것
  2. 특징 : 단일 상속, 중복코드제거,유지보수, 통일성
  
* 메서드오버라이드
  - sub class가 super class에서 정의된 메서드를 재정의하는 것을 의미함.
  - sub class 자원 중 메서드를 서브클래스에서 재정의
  
추상메서드와 추상클래스
: 상속을 통한 재사용성을 높이고, 자식 클래스에서 반드시 구현해야 하는 메서드를 지정하는데 사용됨

추상클래스: - 인스턴스화(객체생성)가  불가능한 클래스.
		 - 추상 클래스는 일반 메서드와 추상 메서드를 모두 가질 수 있음.
		 - 인스턴스를 만들 수 없으므로 상속을 통해서만 사용 가능.

추상메서드: -메서드의 선언만 있고, 구현(메서드의몸체)은 없는 메서드.
        - 이를 상속받은 자식 클래스에서 *반드시* 구현해야함 		 
		 
-------------------------------------------------
다형성
  - 하나의 변수가 다양한 타입의 객체의 주소를 저장
  - 예) 상속 관계에서 슈퍼클래스 변수로 서브클래스
         객체에 접근 가능
up 
is a 관계
   - A는 B이다.   슈퍼클래스를 상속받아 B가 되었다.
